
FSL Spider Chart System ó Database Design Spec
Overview
The Spider Chart system collects per-attribute, per-match comparison votes from a pool of weighted reviewers and aggregates them into a single set of six skill scores per player per division. Those scores drive the radar/"spider" chart displayed on-stream and in the admin dashboard.

Critical semantics for vote:
1 ? player1_id won this attribute
2 ? player2_id won this attribute
0 ? tie (neither player credited a win)
Tie votes contribute neither positive nor negative weight during aggregation ó they're simply excluded from the win/loss counts but do not reduce total_weight (every vote row adds its reviewer's weight to total_weight).
Uniqueness constraint: (reviewer_id, fsl_match_id, attribute) ó one vote per reviewer per match per attribute. Attempting to insert a duplicate will fail.
Indexes: fsl_match_id, reviewer_id, (player1_id, player2_id), attribute, created_at.

Uniqueness constraint: UNIQUE(player_id, division) ó one row per player per division.
Note on score range: Scores are not raw win rates. The aggregation formula is:
win_rate = positive_weighted_votes / total_weighted_votes   (0.0ñ1.0)
normalized = win_rate * 10                                   (0.0ñ10.0)
final_score = (normalized / 2) + 5                          (5.0ñ10.0)
So even a player who loses every attribute vote still gets 5.00, not 0. The chart axis displays 2ñ10 (configured in config.php).

What an External Program Needs to Know
To insert votes (Player_Attribute_Votes):
Resolve Players.Player_ID for both players by matching Players.Real_Name
Resolve fsl_matches.fsl_match_id (and confirm winner_player_id/loser_player_id are correct)
Resolve reviewers.id by reviewers.unique_url token (or name)
Insert one row per attribute (6 total per match review)
vote must be 0, 1, or 2 ó not a score
Constraint (reviewer_id, fsl_match_id, attribute) is unique ó use INSERT Ö ON DUPLICATE KEY UPDATE vote = VALUES(vote) if re-voting is allowed, or check first
To trigger score recalculation after inserting votes:
Run aggregate_scores.php (CLI: php aggregate_scores.php) ó it does a full DELETE + recalculate on Player_Attributes. There is no incremental update path currently.
To read a player's current chart:
SELECT * FROM Player_Attributes WHERE player_id = ? AND division = ?
Divisions in use: 'S', 'A', 'B' (determined by fsl_matches.t_code)
